# DNS Records Reference

This document outlines all the necessary DNS records for the `ternis.net` domain to correctly route traffic to the homelab servers.

### 1. General Web & Reverse Proxy (VPS to mini3)
Because `mini3` acts as the reverse proxy for all web services, all web traffic needs to reach the VPS (`77.90.15.49`) which routes it home via WireGuard.

- **Type**: `A`
- **Name**: `*` (Wildcard)
- **Target**: `77.90.15.49`
- **Purpose**: Routes `immich.ternis.net`, `n8n.ternis.net`, `git.ternis.net`, and any other subdomain automatically to the proxy.

- **Type**: `A`
- **Name**: `@` (or `ternis.net`)
- **Target**: `77.90.15.49`
- **Purpose**: Routes the naked domain to the proxy. **Important**: Without this, Let's Encrypt cannot verify the naked domain and `ternis.net` will not load.

- **Type**: `A`
- **Name**: `www`
- **Target**: `77.90.15.49`
- **Purpose**: Explicit record for the dashboard website (already covered by wildcard, but good practice).

### 2. Mail Server (Poste.io on mx1.ternis.net)
For the mail server to send and receive emails without landing in spam folders, the following DNS records are strictly required.

**Prerequisite:** Port 25 (TCP) must be forwarded directly to `mini1` (`192.168.1.21`) or proxied through `mini3` via a TCP stream.

#### A Record (Mail Host)
- **Type**: `A`
- **Name**: `mx1`
- **Target**: `77.90.15.49`

#### MX Record (Mail Exchanger)
- **Type**: `MX`
- **Name**: `@` (or `ternis.net`)
- **Target**: `mx1.ternis.net`
- **Priority**: `10`

#### SPF Record (Sender Policy Framework)
- **Type**: `TXT`
- **Name**: `@` (or `ternis.net`)
- **Target**: `v=spf1 mx a:mx1.ternis.net -all`

#### DMARC Record
- **Type**: `TXT`
- **Name**: `_dmarc`
- **Target**: `v=DMARC1; p=quarantine; rua=mailto:admin@ternis.net;`

#### DKIM Record (DomainKeys Identified Mail)
*Note: This key is generated by Poste.io after installation.*
- **Type**: `TXT`
- **Name**: `s1._domainkey` (Check the Poste.io admin panel for the exact name)
- **Target**: *(Paste the public key string generated in the Poste.io web interface)*

#### Reverse DNS (PTR Record)
*Note: This cannot be set at your domain registrar. You must set this in the control panel of your VPS provider.*
- **IP**: `77.90.15.49`
- **Target**: `mx1.ternis.net`
